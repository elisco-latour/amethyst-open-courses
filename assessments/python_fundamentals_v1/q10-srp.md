# question
Which statements describe the Single Responsibility Principle (SRP)?

<!-- SEPARATOR -->

# options
- A class should have one, and only one, reason to change
- A class should be open for extension and closed for modification
- A module should encapsulate a single, well-defined responsibility
- Subtypes must be substitutable for their base types

<!-- SEPARATOR -->

# explanation
SRP says a unit of code should have a single reason to change and encapsulate one responsibility. Open/Closed and Liskov Substitution are separate SOLID principles.
